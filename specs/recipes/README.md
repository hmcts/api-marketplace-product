# AMp Recipes

This section captures reusable API Marketplace design patterns.

"Recipe" is the team term for a repeatable implementation pattern with clear trade-offs.
If needed, the same artifacts can also be referred to as design patterns in stakeholder conversations.

## Purpose

- Make proven AMp delivery patterns easy to reuse.
- Keep consistent non-functional expectations (security, reliability, observability).
- Reduce redesign effort across similar API initiatives.

## How to Use

- Start with the recipe template at `specs/recipes/recipe-template.md`.
- Keep recipes technology-agnostic where possible.
- Link each feature spec to one or more recipes when applicable.
